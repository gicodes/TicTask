# Contributing to TicTask


Thank you for your interest in contributing!


## 1. Code Ownership
All contributions are assigned to the owner of TicTask (see OWNERSHIP.md).


## 2. CLA Requirement
You must read and agree to CLA.md before submitting any pull request.


## 3. Branching Strategy
- main → production code
- dev → development branch
- feature/* → new feature branches


## 4. Code Style
- Use consistent naming conventions
- Write clean, commented, maintainable code
- Follow TypeScript + React best practices


## 5. Pull Requests
- Must be linked to an issue
- Must include proper documentation
- Must pass linting and build checks
